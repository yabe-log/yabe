## Code Style
We follow [neostandard](https://github.com/neostandard/neostandard). The highlights are:
- Use single quotes for strings.
- Do not use semicolons at the end of statements.
- Use `camelCase` for variable and function names, and `PascalCase` for class names and React components.
- Use 2 spaces for indentation.

## Linting
We use ESLint to enforce our code style and catch potential issues. The ESLint configuration is defined in `eslint.config.mjs`. Please make sure to run ESLint before committing your code.

## File Naming
- Use `camelCase` for JavaScript/TypeScript files (e.g., `myComponent.tsx`).
- Use `kebab-case` for CSS files (e.g., `my-styles.css`).
- Use `PascalCase` for React components (e.g., `MyComponent.tsx`).

## Commit Messages
- Please follow the [Conventional Commits](https://www.conventionalcommits.org/en/) specification for commit messages. This helps us maintain a clear and consistent commit history.
